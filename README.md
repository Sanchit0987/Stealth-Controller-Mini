# Stealth Controller Mini — SKiDL Schematic

A fully code-defined motor driver board featuring DRV8316 + ESP32 + MA702 + TWAI, written in SKiDL.

---

## Repository Structure

```
stealth-controller-mini/
│
├── driver_skidl.py              # SKiDL schematic source — run this to generate netlists
├── Library.kicad_sym            # Custom KiCad symbol library
├── DriverParts.pretty/          # Custom KiCad footprint library
├── Driver_skidl.net             # Generated netlist (output)
├── Driver_skidl.xml             # Generated XML netlist (output)
│
├── Driver.kicad_pcb             # KiCad PCB layout file
├── Driver.step                  # 3D STEP file of the PCB (exported from KiCad)
├── Driver.stl                   # 3D STL file of the PCB (exported from KiCad)
│
├── convert_netlist.py           # Utility: converts any KiCad .net to SKiDL
├── requirements.txt             # Python dependencies
├── README.md                    # This file
└── .gitignore                   # Files excluded from version control
```

> **Note:** `Driver.step` and `Driver.stl` are the 3D models of the fully laid out PCB.
> These were **not generated by SKiDL** — SKiDL only handles schematic and netlist generation.
> The STEP and STL files were exported from KiCad PCBnew after the full PCB layout was completed
> (component placement + trace routing). Generating a 3D PCB model programmatically from a
> schematic alone is not possible through SKiDL.

## SKiDL Schematic Workflow

This section documents the exact process followed to go from a KiCad schematic file
to a fully code-defined SKiDL Python file.

### Step 1 — Open the schematic in KiCad and identify components

1. Open KiCad and load the project
2. Launch the **Schematic Editor**
3. Go through every component on the schematic and note:
   - **Reference** (e.g. `U1`, `C1`, `IC1`)
   - **Symbol name** (e.g. `DRV8316TRGFR`, `C`, `R`)
   - **Which library it belongs to** (e.g. `Library`, `Device`)
   - **Footprint assigned to it** (e.g. `DriverParts:DRV8316`)
   - **Which nets each pin is connected to**

### Step 2 — Identify the symbol and footprint libraries used

1. In the Schematic Editor go to **Preferences → Manage Symbol Libraries**
   - Note which libraries are listed under **Project Libraries**
   - In this project the custom library is `Library` → `Library.kicad_sym`

2. In KiCad go to **Preferences → Manage Footprint Libraries**
   - Note which footprint libraries are listed under **Project Libraries**
   - In this project the custom footprint library is `DriverParts` → `DriverParts.pretty/`

### Step 3 — Add the symbol library to KiCad global libraries

So SKiDL can find and load symbols from `Library.kicad_sym`:

1. Go to **Preferences → Manage Symbol Libraries → Global Libraries tab**
2. Click the **+** button
3. Set:
   - **Nickname:** `Library`
   - **Library Path:** full path to `Library.kicad_sym` on your machine
   - **Plugin Type:** `KiCad`
4. Click **OK**

### Step 4 — Add the footprint library to KiCad global libraries

So KiCad PCBnew can resolve footprints when importing the generated netlist:

1. Go to **Preferences → Manage Footprint Libraries → Global Libraries tab**
2. Click the **+** button
3. Set:
   - **Nickname:** `DriverParts`
   - **Library Path:** full path to `DriverParts.pretty/` folder on your machine
   - **Plugin Type:** `KiCad`
4. Click **OK**

### Step 5 — Add the symbol library to the project folder

Place `Library.kicad_sym` in the same folder as `driver_skidl.py` so SKiDL
can find it at runtime without any hardcoded absolute paths:

```
stealth-controller-mini/
├── driver_skidl.py
├── Library.kicad_sym    ← must be here
```

SKiDL auto-detects it using:
```python
_lib_dir = os.path.dirname(os.path.abspath(__file__))
for _key in lib_search_paths:
    if _key.startswith("kicad"):
        lib_search_paths[_key].append(_lib_dir)
```

### Step 6 — Export the netlist from KiCad

1. In the Schematic Editor go to **File → Export → Netlist**
2. Select **KiCad** format
3. Click **Export Netlist** and save as `Driver.net`

### Step 7 — Convert the netlist to SKiDL

Use the included converter script to generate `driver_skidl.py` from `Driver.net`:

```bash
python3 convert_netlist.py Driver.net Library.kicad_sym driver_skidl.py
```

This script:
- Parses the KiCad S-expression netlist
- Reads all components, their symbol names, footprints, and net connections
- Verifies every part exists in `Library.kicad_sym`
- Generates a fully structured SKiDL Python file

### Step 8 — Understand the generated SKiDL file structure

The generated `driver_skidl.py` is structured in four sections:

**Power rails** — defined first as special nets:
```python
gnd = Net('GND'); gnd.drive = POWER
vm  = Net('VM');  vm.drive  = POWER
```

**Signal nets** — one variable per net in the schematic:
```python
spi_mosi = Net('SPI_MOSI')
spi_miso = Net('SPI_MISO')
i2c_scl  = Net('I2C_SCL')
```

**Parts** — each component loaded from `Library.kicad_sym` with its footprint:
```python
U1 = Part('Library', 'DRV8316TRGFR',
        footprint='DriverParts:DRV8316',
        tag='U1')
U1.ref   = 'U1'
U1.value = 'DRV8316R'
```

**Connections** — each net connects to the pins of every component on that net:
```python
gnd += C1['2'], C2['2'], U1['GND'], IC1['GND']
spi_mosi += IC1['IO13_15'], U1['SDI']
```

### Step 9 — Add components one by one with connected footprints

When adding parts manually (instead of using the converter), each part follows this pattern:

```python
# Syntax: Part('library_nickname', 'symbol_name', footprint='LibNickname:footprint_name')
C1 = Part('Library', 'C',
        footprint='DriverParts:C_0402',
        tag='C1')
C1.ref   = 'C1'
C1.value = '100nF'
```

Key rules:
- `'Library'` must match the **nickname** registered in KiCad symbol libraries
- `'DriverParts:...'` must match the **nickname:filename** in KiCad footprint libraries
- `tag` should match the reference for traceability

### Step 10 — Create nets and form connections

Define every net that appears in your schematic:

```python
# Power
vcc = Net('+3.3V'); vcc.drive = POWER
gnd = Net('GND');   gnd.drive = POWER

# Signals
spi_clk  = Net('SPI_CLK')
spi_mosi = Net('SPI_MOSI')
```

Then connect pins to nets using `+=`:

```python
# Connect multiple pins to the same net in one line
gnd += C1['2'], C2['2'], R1['2'], U1['GND']

# Or one at a time
vcc += C1['1']
vcc += U1['VCC']
```

Pin numbers or names from the symbol can both be used:
```python
U1['GND']        # by pin number
U1['VCC_3V3']    # by pin name
```

## Generate Netlist

Once `Library.kicad_sym` is in the project folder, run:

```bash
python3 driver_skidl.py
```

This generates:
- `Driver_skidl.net` — KiCad netlist, import via **PCBnew → File → Import Netlist**
- `Driver_skidl.xml` — KiCad XML netlist for KiCad 6+

---

## Import into KiCad PCBnew

1. Open KiCad → open your project
2. Launch **PCBnew**
3. Go to **File → Import Netlist**
4. Select `Driver_skidl.net`
5. Click **Update PCB**

All components will appear with their footprints ready for placement and routing.

## 3D Files (STEP and STL)

The `Driver.step` and `Driver.stl` files in this repo are the 3D models of the
fully completed PCB after layout and routing in KiCad PCBnew.

**These are not generated by SKiDL.** The workflow to produce them was:

```
driver_skidl.py  →  Driver_skidl.net  →  KiCad PCBnew (place + route)  →  Export STEP/STL
```

To export them yourself after completing the PCB layout in KiCad:
- **STEP:** PCBnew → **File → Export → STEP**
- **STL:** PCBnew → **File → Export → STL**

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `Can't open file: Library` | Make sure `Library.kicad_sym` is in the same folder as `driver_skidl.py` |
| `fp-lib-table not found` | Register `DriverParts.pretty/` in KiCad Footprint Libraries (Step 4 above) |
| `KICAD9_SYMBOL_DIR missing` | Set env var to KiCad symbols dir or register library globally (Step 3 above) |
| Footprints missing in PCBnew | Register `DriverParts.pretty/` in KiCad global footprint libraries |
| Part not found in library | Verify symbol name matches exactly what's in `Library.kicad_sym` |
